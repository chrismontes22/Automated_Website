Amazon S3 server access logs now deliver to Amazon CloudWatch Logs and Amazon S3 Tables
============================================================

Amazon S3 now supports delivering server access logs to Amazon CloudWatch Logs and Amazon S3 Tables, providing users with enhanced monitoring, real-time querying, and improved data security capabilities.

### Key Points
- Amazon S3 server access logs can now be delivered directly to Amazon CloudWatch Logs for real-time monitoring and alerting.
- Logs can be mirrored to Amazon S3 Tables in Apache Iceberg format without incurring additional storage costs.
- Integration with CloudWatch Logs enables cross-account and cross-Region log aggregation and AWS KMS encryption.
- Data stored in S3 Tables is immediately queryable using SQL via Amazon Athena, Amazon Redshift, and other Iceberg-compatible engines.
- The new delivery options are available in all AWS Regions, excluding AWS China and AWS GovCloud (US) locations.

### Why it Matters
These updates provide organizations with more flexible tools to audit security, track usage trends, and troubleshoot operational issues across their cloud infrastructure. By streamlining log analysis, businesses can more efficiently manage data access patterns and reduce the time required to investigate potential security incidents.
