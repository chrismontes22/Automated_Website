Anthropic/OpenAI may be spending more than $1000 for every $100 you pay them
============================================================

Coding with Large Language Models has become a powerful tool for software development, but the hidden costs of recursive "thinking" models are significantly higher than many users realize.

### Key Points
- Using frontier models like Claude Opus 4.6 for complex, multi-file coding tasks can cost up to $65 per task when billed at standard API rates.
- "Thinking" or recursive models generate massive amounts of invisible background tokens, which are billed at the higher output token rate of $25 per million.
- Subscription plans often heavily subsidize these costs, with some users receiving up to 12 times the value of their monthly fee in actual API-level token usage.
- Coding tasks are "fault-intolerant," requiring high-effort settings that consume significantly more computational resources than simple, fault-tolerant conversational queries.
- The current economic model relies on heavy subsidies that may become unsustainable as companies approach IPOs and seek to improve profitability.

### Why it Matters
The reliance on "brute force" recursive coding creates a temporary productivity boom that masks the true, high cost of AI-assisted software development. Businesses must prepare for a future where these heavy subsidies disappear, potentially forcing a reevaluation of whether AI-generated code remains economically viable for complex, large-scale projects.
