Godot bans "vibe-coded" code contributions
============================================================

The Godot game engine development team is updating its contribution policy to strictly prohibit AI-generated code, citing concerns over code quality and the inability of contributors to maintain submissions.

### Key Points
- Godot maintainers are banning autonomous AI-authored code and "vibe-coded" contributions to prevent repository clutter.
- New contributors with three or fewer merged pull requests must now obtain explicit permission before submitting significant features or refactoring.
- AI usage is restricted to menial tasks like code completion or regex, and any AI assistance must be disclosed in pull request discussions.
- Communication channels will be restricted to human-to-human interaction, explicitly banning AI agents and bots from participating in discussions.
- The team stated that AI-generated pull requests are demoralizing and create an unsustainable burden for maintainers who must review and fix the code.

### Why it Matters
This policy shift highlights a growing tension between open-source maintainers and the influx of automated code contributions that lack human oversight. By prioritizing code comprehension and accountability, Godot is setting a precedent for how major software projects may restrict AI tools to preserve long-term project stability and developer morale.
