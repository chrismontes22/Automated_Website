Indie Game Engine Will Require ‘All Code To Be Human-Authored’ Following An AI Code ‘Shitshow’
============================================================

The Godot game engine has implemented a strict new contribution policy requiring all code and communication to be human-authored to combat a surge of AI-generated pull requests.

### Key Points
- Godot’s GitHub repository currently faces a backlog of over 5,000 unresolved pull requests, largely attributed to AI-generated submissions.
- The updated policy mandates that all code must be human-authored, explicitly banning "vibe coding" and machine-generated project descriptions.
- Developers may still use AI for menial tasks like find-and-replace or code autocompletion, provided they include a formal disclosure.
- Project manager Rémi Verschelde noted that AI contributions demoralize maintainers by removing the opportunity to mentor human developers.
- The policy change aims to restore efficiency and respect for the volunteer maintainers who review community submissions.

### Why it Matters
This policy shift highlights the growing tension between open-source project sustainability and the influx of automated content. By prioritizing human interaction, Godot aims to preserve its community-driven development model while distancing itself from the industry trend of integrating generative AI tools.
